# Лабораторна робота №2: Аналіз та обробка даних за допомогою Pandas

Ця директорія містить файли другої лабораторної роботи. Робота присвячена поглибленому вивченню бібліотеки Pandas: завантаженню, очистці, складному фільтруванню великих масивів даних та їх підготовці до машинного навчання.

## Структура роботи

Робота розділена на два Jupyter Notebook файли:

### 1. `Part1.ipynb` — Робота з індексами здоров'я рослинності (VHI)
У цій частині реалізовано автоматизований пайплайн для роботи з супутниковими даними NOAA:
* Автоматичне завантаження `.csv` файлів для кожної області України.
* Очищення даних (Data Cleaning), заміна "сирих" індексів областей на стандартизовані українські назви.
* Реалізація функцій для вибірки даних:
  * Ряд VHI для заданої області та конкретного року.
  * Ряд VHI за вказаний діапазон років для списку заданих областей.
* Статистичний аналіз за допомогою `.groupby()`: розрахунок екстремумів (min, max), середнього (mean) та медіани (median) для вказаних областей та років.

### 2. `Part2.ipynb` — Аналіз споживання електроенергії
Робота з масивним датасетом *Individual Household Electric Power Consumption* (понад 2 млн записів):
* Автоматичне завантаження архіву, його розпакування та зчитування даних у пам'ять.
* Очистка даних: обробка пропусків (заміна `?` на `NaN` та їх видалення), правильне форматування типів `datetime`.
* Створення складних вибірок із заміром часу виконання кожної функції за допомогою модуля `timeit`:
  * Фільтрація записів за активною потужністю (> 5 кВт).
  * Багатокритеріальна фільтрація: сила струму (19-20 А) та порівняння груп споживання приладів.
  * Випадкова вибірка 500 000 записів (без повторень) та розрахунок середніх величин.
  * Складний часовий та логічний фільтр (після 18:00, > 6 кВт, домінування 2-ї групи споживання) з кастомним зрізом рядків (кожен 3-й та 4-й елемент з поділеного навпіл датафрейму).
* Статистика та підготовка даних:
  * Розрахунок кореляції Пірсона та Спірмена для числових ознак.
  * Стандартизація даних (Z-score).
  * Створення нової категоріальної змінної (день тижня) та виконання One-Hot Encoding.
